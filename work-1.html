Overview

Fleet operators managing dozens or hundreds of vehicles face a daily problem: correctly assigning which driver is using which vehicle. Without accurate mapping, telematics data becomes unreliable — hours of service logs, driver behaviour scores, and compliance reports all depend on knowing who was behind the wheel.

This project delivered an AI/ML system that learns driver-vehicle patterns from historical trip data and automatically assigns the most probable driver to each vehicle at the start of every trip — achieving 90% prediction accuracy and removing the need for manual assignment across targeted fleets.

The Problem

Manual assignment is error-prone

Operators manually logging driver-vehicle pairings introduced frequent errors, corrupting downstream compliance and behaviour reports.

Daily operational overhead

Fleet managers spent significant time each morning assigning drivers — time that should go to managing exceptions, not routine data entry.

Unreliable telematics data

Misattributed trips skewed driver behaviour scores, invalidated HOS logs, and undermined the trust operators placed in the platform.

Solution & Architecture

The system analyses historical trip patterns — time of day, start location, route signature, and vehicle preferences — to build a probabilistic model per driver. At the moment a trip begins, the model scores each known driver against the current trip context and assigns the highest-confidence match, with a fallback to manual assignment when confidence is below threshold.

Key Features

  • Automatic trip assignment — drivers mapped to vehicles at trip start without any manual input required from fleet operators
  • Confidence scoring — every prediction includes a confidence score; low-confidence trips are flagged for manual review rather than silently misassigned
  • Manual override with learning — operators can correct any assignment, and each correction strengthens the model for that driver-vehicle context
  • Fleet-level accuracy dashboard — operators can see prediction accuracy trends, correction rates, and model confidence distribution across their fleet
  • Graceful degradation — when a driver is new or has insufficient history, the system surfaces manual assignment prompts rather than making a low-quality prediction

Impact & Outcomes

The system reached 90% prediction accuracy across targeted fleet types within the initial rollout period, dramatically reducing the volume of manual assignments fleet operators needed to make each day. For large fleets, this translated to hours of recovered operator time per week and a significant improvement in the reliability of downstream telematics reports.

The product approach — starting with high-confidence fleet segments and expanding gradually — was key to building operator trust. By showing accurate predictions before asking operators to rely on them fully, adoption was significantly smoother than a big-bang rollout would have been.

Lessons Learned

  • Data quality upstream is the real constraint — the model was only as good as the historical assignment data it trained on. Investing in data cleaning and validation before model training saved significant rework downstream.
  • Explainability matters as much as accuracy — fleet managers wanted to understand why a driver was assigned, not just that the model said so. Adding simple reasoning text ("assigned based on usual morning route") dramatically improved trust and adoption.
  • Gradual rollout beats big-bang deployment — starting with high-confidence predictions only and expanding the automation threshold over time gave operators a chance to build confidence in the system before fully relying on it.
  • The feedback loop is the product's long-term moat — every correction makes the model better for that fleet. Operators who engaged with the feedback UI saw faster accuracy improvements, making the feedback mechanism a retention driver, not just a quality tool.
